How did you implement the USER ENTITY to the system you created? 
Give THE COMPLETE STEPS BELOW with some screenshots of the steps you took.

ANSWER:

I implemented the USER ENTITY by merging it into the existing accounts table. 
Instead of creating a separate users table, I added the login/register 
functionality directly into the accounts table so it handles both 
user authentication and storing personal information. Below are the steps:


STEP 1: Updated the accounts table in data.sql to include a password column 
for login, and added_by/updated_by columns to track who added or updated 
each record.

    CREATE TABLE accounts (
        account_id INT AUTO_INCREMENT PRIMARY KEY,
        username VARCHAR(50),
        email VARCHAR(100),
        password VARCHAR(255),
        added_by VARCHAR(50),
        updated_by VARCHAR(50),
        date_registered TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    );

each record.

STEP 4: Created user functions in models.php:
- insertNewUser() - checks if username already exists, if not, inserts 
  new account with username, email, and hashed password into the accounts table.
- loginUser() - verifies username and password from the accounts table 
  and sets the session.
- getAllAccounts() and getAccountByID() - fetches account records.

STEP 5: Updated all insert and update functions in models.php to accept 
added_by and updated_by parameters. For example:

    function insertGuitar($pdo, $guitar_name, $guitar_type, $quantity, $price, $added_by)

    function updateGuitar($pdo, $guitar_name, $guitar_type, $quantity, $price, $guitar_id, $updated_by)

This way, every time a record is inserted or updated, we store the username 
of the logged-in user who performed that action.

STEP 6: Created register.php and login.php pages for user registration 
and login. The register page collects username, email, and password. 
The password is hashed using sha1() before storing in the database.

STEP 7: Added form handlers in handleForms.php for register, login, 
and logout. The register handler passes the email along with username 
and hashed password. The login handler verifies credentials and creates 
a session. The logout handler destroys the session.

STEP 8: Updated all existing form handlers to pass $_SESSION['username'] 
as added_by when inserting records, and as updated_by when updating records. 
For example:

    insertGuitar($pdo, ..., $_SESSION['username']);
    updateGuitar($pdo, ..., $_SESSION['username']);

STEP 9: Added session protection to all pages. If a user is not logged in, 
they get redirected to login.php:

    if (!isset($_SESSION['username'])) {
        header("Location: login.php");
    }

STEP 10: Updated all tables and cards in the pages to display the 
"Added By" and "Updated By" columns so users can see who added

SUMMARY:
The USER ENTITY was implemented by using the accounts table for both 
registration/login and storing user information. Every page is protected 
by session checking. The added_by and updated_by columns in all tables 
track which logged-in user performed each action. This way, we always 
know who added and who last updated a certain guitar record, other product 
record, or shopping cart item.
